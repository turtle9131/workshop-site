Group B · Planning

B그룹 · 기획서형 실습

시장자료·고객 문제·경쟁 서비스 자료를 기반으로, AI를 활용해 문제 정의 → 솔루션 콘셉트 → 핵심 기능 → MVP 범위 → 기획서 목차를 도출하고 기획서 초안을 완성합니다.

주요 대상: 솔루션기획 · 신사업 · 서비스기획 · 개발리더 · PM 산출물: 신규 솔루션 기획서 초안
📋

샘플 과제

AI 기반 회의록·업무보고 자동화 솔루션 기획 — 개인 준비 주제가 있다면 아래 자료 대신 본인 주제를 사용하세요.

자료내용
시장 트렌드 자료생성형 AI 업무 자동화 동향
고객 페인포인트회의록, 보고서, To-do 관리 문제
경쟁 서비스 자료AI 회의록, 업무 자동화 SaaS 사례
내부 역량 자료SI, 클라우드, 운영, AI 기술 역량
기획서 템플릿문제 정의, 타깃 고객, 기능, 로드맵 양식
준비 자료가 없다면, 각 단계 프롬프트를 그대로 복사해 붙여넣어도 실습을 진행할 수 있도록 예시가 포함되어 있습니다.
01

1교시 · 시장·고객 문제 분석

시장자료와 고객 문제를 분석해 솔루션 기회와 사업화 포인트를 도출합니다.

B1

NotebookLM에 자료 업로드

NotebookLM 00:00~00:50
  1. NotebookLM 접속 → 새 노트북 만들기
  2. 시장 트렌드·고객 페인포인트·경쟁 서비스 자료를 소스로 추가
  3. 아래 프롬프트로 시장·고객 문제 분석표를 생성합니다.
프롬프트 · 시장·고객 문제 분석표 
업로드된 시장·고객·경쟁 자료를 분석해 아래 표 형식으로 정리해줘.

1. 시장 변화: 이 영역에서 지금 일어나고 있는 핵심 변화
2. 고객 페인포인트: 타깃 고객이 실제로 겪는 문제 5가지
3. 문제의 심각도/빈도: 각 페인포인트가 얼마나 자주·크게 발생하는지
4. 경쟁 서비스 현황: 기존 서비스가 해결하는 부분과 못 하는 부분
5. 해결 기회(Gap): 아직 충분히 해결되지 않은 빈틈

각 항목마다 어느 자료에 근거했는지 출처를 함께 표시해줘.
프롬프트 · 추가 질의(선택) 
위 분석에서, 우리가 가장 우선적으로 공략해야 할 고객 문제 1가지를 고르고
그 이유(시장성·해결 가능성·차별성)를 근거와 함께 설명해줘.

✅ 이 단계 산출물

  • 시장·고객 문제 분석표 (시장 변화·페인포인트·경쟁·기회)
  • 우선 공략 문제 1가지
02

2교시 · 기획서 초안 작성

분석 결과를 신규 솔루션 기획서 초안으로 변환합니다. NotebookLM 분석표를 복사해 Gemini에 붙여넣으세요.

B2

Gemini + Canvas로 콘셉트·초안 작성

Gemini + Canvas 01:00~01:50
  1. Gemini 접속 → 입력창에서 Canvas 기능 켜기
  2. 1교시 분석표를 붙여넣고 아래 프롬프트로 솔루션 콘셉트를 도출합니다.
  3. 이어서 초안 작성 프롬프트로 Canvas 문서를 생성합니다.
프롬프트 · 솔루션 콘셉트 & 핵심 기능 
아래는 시장·고객 문제 분석 결과야.
[여기에 1교시 분석표 붙여넣기]

이 분석을 바탕으로 신규 솔루션을 기획해줘.
1. 솔루션 콘셉트: 어떤 문제를 어떤 방식으로 해결하는지 한 문장 정의
2. 타깃 고객: 1차 타깃과 사용 상황
3. 핵심 기능 목록: 기능명 / 고객 가치 / 우선순위(상·중·하)를 표로
4. 차별점: 경쟁 서비스와 다른 점 3가지
표와 짧은 설명을 함께 정리해줘.
프롬프트 · 기획서 초안 (Canvas) 
위 솔루션 콘셉트와 문제 분석을 반영해서, 신규 솔루션 기획서 초안을 Canvas에 작성해줘.
아래 구성으로 각 장마다 3~5문장 분량의 초안을 작성해:

1. 배경 및 문제 정의
2. 솔루션 콘셉트 및 타깃 고객
3. 핵심 기능 (기능별 가치)
4. MVP 범위 (초기 검증용 최소 기능)
5. 차별화 포인트
6. 기대효과 및 추진 방향

경영진 보고용 기획서 톤(간결하고 설득력 있는 문체)으로 작성해줘.

✅ 이 단계 산출물

  • 솔루션 콘셉트 + 핵심 기능 목록(우선순위 포함)
  • Canvas 기획서 초안 (6개 장 구성)
03

3교시 · 논리 구조화와 산출물 완성

경영진이 판단하기 쉬운 사업화 스토리라인과 기획서 목차를 완성합니다.

B3

Claude로 MVP·목차·발표 요약 완성

Claude 02:00~02:50
  1. Claude 접속 → 새 대화 시작
  2. 2교시 Canvas 초안을 붙여넣고 아래 프롬프트로 MVP 범위와 목차를 정교화합니다.
  3. 마지막으로 3분 발표 요약을 생성합니다.
프롬프트 · MVP 범위 + 기획서 목차 
아래는 신규 솔루션 기획서 초안이야.
[여기에 2교시 기획서 초안 붙여넣기]

두 가지를 정리해줘.
1. MVP 범위: 초기 3개월 안에 검증 가능한 최소 기능만 골라
   "포함 / 제외 / 그 이유"를 표로
2. 기획서 목차: 경영진 보고용 기획서 장·절 구성(1., 1.1 형식)으로,
   의사결정 흐름(문제→솔루션→검증→기대효과) 순서로

MVP 표와 목차를 함께 보여줘.
프롬프트 · 3분 발표 요약 
이 기획 내용을 경영진 앞에서 3분간 보고한다고 가정하고,
발표 스크립트를 작성해줘.
- 문제 정의 (30초)
- 솔루션 콘셉트와 핵심 기능 (90초)
- MVP 추진 방향과 기대효과 (60초)
실제로 읽을 수 있는 구어체 문장으로 작성해줘.

✅ 최종 산출물

  • 시장·문제 분석표 · 솔루션 콘셉트 · 기획서 초안
  • MVP 범위표 + 기획서 목차
  • 3분 발표 스크립트
마무리(02:50~03:00): 완성한 기획서 초안을 팀과 공유하고, 실제 솔루션 기획 업무에 어떻게 적용할지 한 가지씩 정리해 보세요.
