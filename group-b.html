역할 정의 입력값

프롬프트 · 핵심 주제 도출

| 중요도(1~5순위) | 핵심 주제 및 주요 관점 | 대표 소스 제목 | 선정 이유 |

② 교차 검증·신뢰성 평가 (Cross-Validation)

프롬프트 · 신뢰성 검증 표 
업로드한 자료들의 신뢰도와 객관성을 꼼꼼하게 검증하려고 해. 제공된 모든 소스를 검토해서 아래 양식에 따라 표로 깔끔하게 정리해 줘.

| 파일명 | 소스 제목 | 발행일 | 저자(소속) | 소스 구분 | 핵심 주제 및 관점 | 주장을 뒷받침하는 주요 근거 |

③ 인사이트 도출·리스크 분석 (Insight & Analysis)

프롬프트 ③-1 · 공통 인사이트 
업로드된 모든 문서를 교차 분석하여, 서로 다른 소스 간에 공통적으로 연결되는 핵심 인사이트 3가지를 도출해 줘.
프롬프트 ③-2 · 액션 아이템

질문 1 · 시장 배경 
업로드된 자료를 바탕으로, 바이오 연구 기관들이 왜 자체 서버 대신 클라우드 기반의 TRE(Trusted Research Environment) 플랫폼을 도입해야 하는지 핵심 이유 3가지를 정리해 줘.
질문 2 · 보안/규제 대책 
유전체 데이터 분석 시 법적 규제를 준수하기 위해 플랫폼이 반드시 갖춰야 할 보안 필수 요구사항을 요약해 줘. 답변 내용에 해당하는 소스 출처(Citation) 번호도 함께 확인해 봐.
질문 3 · 기술 아키텍처 요구사항 
이 플랫폼의 인프라 구성을 위해 필요한 네트워크 분리(DMZ/내부망), API 게이트웨이, L4 로드밸런서, 그리고 Kubernetes 노드 구성 요건을 표(Table) 형태로 정리해 줘.
💡 메모 → 소스로 저장: 채팅으로 얻은 답변은 [메모에 복사]로 저장한 뒤, 그중 특히 쓸모 있는 메모는 해당 메모를 선택해 [소스로 저장(변환)] 하세요. 메모가 노트북의 정식 소스가 되어 이후 분석·검증과 다음 단계에서 근거로 재활용됩니다.

Claude로 발표용 PPT(.pptx) 만들기

Claude · Design 02:20~02:45

슬라이드 구성이 정해지면, Claude 대화에 필요한 자료를 첨부하고 아래 상세 프롬프트로 다운로드 가능한 PowerPoint(.pptx)를 생성합니다.

  1. Claude 데스크톱 앱 대화에 다음을 첨부합니다.
    • 2교시에서 저장한 기획서 초안(.md) 파일

    • McKinsey Consulting Proposal.pdf — 표지·간지·내용·도식 디자인 및 논리 전개 참고용 · 사전 준비에서 다운로드
  2. 아래 프롬프트를 붙여넣어 PPT 생성을 요청합니다.
Claude 프롬프트 ② · 발표용 PPT 생성 
첨부한 '유전체 분석 TRE 플랫폼 신사업 기획서 초안(.md)' 파일 내용을 참고해서 발표용 PPT를 만들어줘.

1. 기획서 내용을 슬라이드로 나누어 A4 비율(4:3)의 PPT로 만들어줘. 표지 → 사업 배경·필요성 → 글로벌 시장 트렌드 → 솔루션·보안 아키텍처 → 비즈니스 모델 → 기대효과 → 다음 단계 순서로, 약 8~10장으로 구성해줘.
2. 컬러는 #FFFFFF, #161A1F, #2768FF, #1B2E59, #FFFF68 활용. 필요에 따라 유사색 활용해줘.
3. 폰트는 제목용 폰트 'Pretendard SemiBold', 본문용 폰트 'Pretendard Regular' 활용(폰트 첨부 완료).
4. 심플한 아이콘, 도형 위주의 디자인.
5. 줄글을 내용의 흐름에 맞춰 도식화. 특히 솔루션 아키텍처는 DMZ/내부망 분리, API 게이트웨이, L4 로드밸런서, Kubernetes 노드를 도형과 화살표로 도식화해줘.
6. 첨부한 'McKinsey Consulting Proposal.pdf'를 디자인·구성 레퍼런스로 삼아 슬라이드 유형별로 참고해줘:
   - 표지 슬라이드 → PDF의 표지 레이아웃(타이틀 배치·여백·톤앤매너) 참고
   - 섹션 구분(간지) 슬라이드 → PDF의 섹션 전환 페이지 스타일 참고
   - 일반 내용 슬라이드 → PDF의 본문 슬라이드 구성(헤드라인·불릿·여백) 참고
   - 차트/데이터·아키텍처 슬라이드 → PDF의 도식·다이어그램 표현 방식 참고
   아울러 PDF의 논리 전개 방식(문제 정의 → 접근법 → 솔루션 → 기대효과 흐름)도 우리 기획서 내용에 맞게 벤치마킹해줘.
7. 완성된 .pptx 파일을 내려받을 수 있게 해줘.

내려받은 .pptx는 PowerPoint에서 열어 색상·문구를 최종 손질한 뒤 그대로 발표할 수 있습니다.

Claude와 대화하며 PPT 다듬기

3교시에서 생성된 PPT가 마음에 들지 않으면, Claude에게 추가 피드백을 주며 고도화해 보세요. 요청할 때마다 .pptx가 다시 생성됩니다. 아래처럼 PowerPoint 안에 Claude 애드인을 설치하면, 파일을 오가지 않고 PowerPoint 화면에서 바로 대화하며 슬라이드를 다듬을 수 있습니다.

🧩 PowerPoint용 Claude 애드인 설치
① PowerPoint(데스크톱 앱)에서 3교시에 생성한 .pptx 파일을 엽니다.
② 상단 리본의 [삽입] 탭 → [추가 기능 가져오기](Get Add-ins)를 클릭합니다.
③ Microsoft AppSource 검색창에 "Claude"를 검색합니다.
④ 검색된 Claude 애드인을 선택해 [추가]를 눌러 설치합니다.
⑤ 설치가 끝나면 리본 메뉴에 Claude 아이콘이 생기고, 클릭하면 화면 오른쪽에 Claude 사이드바가 열립니다.
⑥ 사이드바에서 Claude 계정으로 로그인하면 바로 대화를 시작할 수 있습니다.
💡 애드인으로 PPT 다듬는 법: 다듬고 싶은 슬라이드를 PowerPoint에서 선택한 상태로 사이드바에 수정 요청을 입력하면, Claude가 현재 열려 있는 프레젠테이션을 직접 인식해 해당 슬라이드에 반영할 변경안을 제안합니다. 제안 내용을 확인한 뒤 적용하면 별도로 파일을 내려받고 다시 여는 과정 없이 바로 슬라이드에 반영됩니다. (조직의 Claude 플랜에 따라 애드인 사용 가능 여부가 다를 수 있으니, 보이지 않으면 진행자에게 문의하세요.)
