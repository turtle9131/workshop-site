Group A · Proposal

A그룹 · 제안서형 실습

공공기관 SI 사업의 샘플 RFP를 기반으로, AI를 활용해 요구사항 분석 → 제안 전략 → 제안서 목차 → 핵심 메시지를 도출하고 제안서 초안을 완성합니다.

주요 대상: 사업기획 · 영업 · 제안 · PM · 기술제안 산출물: 공공기관 SI 제안서 초안
📋

샘플 과제

공공기관 통합 업무포털 고도화 사업 제안서 작성 — 개인 준비 과제가 있다면 아래 자료 대신 본인 자료를 사용하세요.

자료내용
샘플 RFP사업 배경, 추진 목적, 과업 범위
요구사항 목록기능 요구사항, 비기능 요구사항
평가 기준표기술평가 항목, 제안서 작성 기준
기관 현황 요약기존 시스템 문제점, 사용자 불편사항
유사 사업 사례포털 고도화, 행정시스템 개선 사례
준비 자료가 없다면, 각 단계 프롬프트를 그대로 복사해 붙여넣어도 실습을 진행할 수 있도록 예시가 포함되어 있습니다.
01

1교시 · 자료 기반 분석

RFP를 읽고 사업 배경, 요구사항, 평가 포인트, 제안 리스크를 구조화합니다.

A1

NotebookLM에 자료 업로드

NotebookLM 00:00~00:50
  1. NotebookLM 접속 → 새 노트북 만들기
  2. 샘플 RFP·요구사항 목록·평가 기준표 파일(PDF/문서)을 소스로 추가
  3. 아래 프롬프트를 붙여넣어 RFP 분석표를 생성합니다.
프롬프트 · RFP 분석표 
업로드된 RFP와 요구사항 자료를 분석해 아래 표 형식으로 정리해줘.

1. 사업 배경: 발주기관이 이 사업을 추진하는 이유와 현재 문제점
2. 추진 목적 / 기대효과
3. 과업 범위: 핵심 과업을 항목별로 요약
4. 기능 요구사항 / 비기능 요구사항 (각각 5개 내외)
5. 평가 포인트: 기술평가에서 배점이 높거나 강조된 항목
6. 제안 리스크: 요구사항이 모호하거나 난이도가 높은 부분

각 항목마다 RFP의 어느 부분에 근거했는지 출처를 함께 표시해줘.
프롬프트 · 추가 질의(선택) 
평가위원 관점에서 이 RFP에서 가장 중요하게 볼 핵심 요구사항 3가지와,
그 이유를 근거와 함께 알려줘.

✅ 이 단계 산출물

  • RFP 분석표 (사업 배경·요구사항·평가 포인트·리스크)
  • 핵심 요구사항 Top 3
02

2교시 · 제안서 초안 작성

분석 결과를 공공기관 SI 제안서 초안으로 변환합니다. NotebookLM 분석표를 복사해 Gemini에 붙여넣으세요.

A2

Gemini + Canvas로 제안 전략·초안 작성

Gemini + Canvas 01:00~01:50
  1. Gemini 접속 → 입력창에서 Canvas 기능 켜기
  2. 1교시에서 만든 RFP 분석표를 붙여넣고 아래 프롬프트로 제안 전략을 도출합니다.
  3. 이어서 초안 작성 프롬프트로 Canvas 문서를 생성합니다.
프롬프트 · 제안 전략 
아래는 공공기관 SI 사업의 RFP 분석 결과야.
[여기에 1교시 RFP 분석표 붙여넣기]

이 분석을 바탕으로 제안 전략을 도출해줘.
1. 제안 방향 (한 문장 핵심 컨셉)
2. 경쟁사 대비 차별화 포인트 3가지
3. 수행 전략 (사업 관리·품질·리스크 관리 관점)
4. 발주기관이 얻게 될 핵심 가치
표와 짧은 설명을 함께 정리해줘.
프롬프트 · 제안서 초안 (Canvas) 
위 제안 전략과 RFP 분석을 반영해서, 공공기관 SI 제안서 초안을 Canvas에 작성해줘.
아래 구성으로 각 장마다 3~5문장 분량의 초안을 작성해:

1. 사업 이해 및 추진 배경
2. 제안 개요 및 제안 방향
3. 요구사항 대응 방안 (기능/비기능)
4. 차별화 전략 및 강점
5. 사업 수행 및 관리 방안
6. 기대효과

공공기관 제안서 톤(격식 있고 명확한 문체)으로 작성해줘.

✅ 이 단계 산출물

  • 제안 전략표 (제안 방향·차별화·수행 전략)
  • Canvas 제안서 초안 (6개 장 구성)
03

3교시 · 논리 구조화와 산출물 완성

평가위원이 이해하기 쉬운 제안 스토리라인과 제안서 목차를 완성합니다.

A3

Claude로 목차·핵심 메시지·발표 요약 완성

Claude 02:00~02:50
  1. Claude 접속 → 새 대화 시작
  2. 2교시 Canvas 초안을 붙여넣고 아래 프롬프트로 목차와 핵심 메시지를 정교화합니다.
  3. 마지막으로 3분 발표 요약을 생성합니다.
프롬프트 · 제안서 목차 + 핵심 메시지 
아래는 공공기관 SI 제안서 초안이야.
[여기에 2교시 제안서 초안 붙여넣기]

두 가지를 정리해줘.
1. 제안서 목차: 공공기관 제안서용 장·절 구성(1., 1.1 형식)으로, 평가 기준에 맞춘 순서로
2. 핵심 메시지: 각 장마다 평가위원 관점에서 "이 부분에서 반드시 각인시킬 한 문장"

목차와 핵심 메시지를 표로 나란히 보여줘.
프롬프트 · 3분 발표 요약 
이 제안 내용을 평가위원 앞에서 3분간 발표한다고 가정하고,
발표 스크립트를 작성해줘.
- 사업 이해 (30초)
- 제안 핵심과 차별화 (90초)
- 기대효과와 마무리 (60초)
실제로 읽을 수 있는 구어체 문장으로 작성해줘.

✅ 최종 산출물

  • RFP 분석표 · 제안 전략 · 제안서 초안
  • 제안서 목차 + 장별 핵심 메시지
  • 3분 발표 스크립트
마무리(02:50~03:00): 완성한 제안서 초안을 팀과 공유하고, 실제 제안 업무에 어떻게 적용할지 한 가지씩 정리해 보세요.
